## 2. Evaluation Scheme

The course grading is heavily weighted toward practical application and the final examination.

- **Labs/Practices:** 10%

- **Group Project (BTL):** 40%

- **Final Exam:** 50% (90-minute Multiple Choice Question format)

---

## 3. Learning Outcomes (LOs)

The course aims to develop three primary competencies:

- **LO1:** Understand basic concepts of software management, including project elements and management contents.

- **LO2:** Use specialized tools (such as Azure DevOps or Microsoft Project) to manage software and analyze tool-generated results.

- **LO3:** Demonstrate effective teamwork and communication skills.

---

## 4. Course Content & Weekly Schedule

The course is divided into 12 main chapters covering the project lifecycle:

| Session   | Topic        | Key Content                                                     |
| --------- | ------------ | --------------------------------------------------------------- |
| **Ch. 1** | Introduction | Project vs. software project, stakeholders, and business cases. |

|
| **Ch. 2** | Evaluation | Project portfolio management and cost-benefit techniques.

|
| **Ch. 3** | Planning Overview | Step-wise planning, scope identification, and resource allocation.

|
| **Ch. 4** | Project Approach | Selection of process models (Waterfall, Spiral, Agile, XP).

|
| **Ch. 5** | Effort Estimation | Bottom-up/top-down methods, Function Point Analysis, and COCOMO.

|
| **Ch. 6** | Activity Planning | Network planning, Forward/Backward pass, and **Critical Path**.

|
| **Ch. 7** | Risk Management | Risk identification, planning, PERT, and Monte Carlo simulation.

|
| **Ch. 8** | Resource Allocation | Scheduling resources and publishing cost schedules.

|
| **Ch. 9** | Monitoring & Control | Data collection, Earned Value Analysis, and change control.

|
| **Ch. 10** | Managing People | Motivation (Oldham-Hackman model), stress, and health/safety.

|
| **Ch. 11** | Working in Teams | Decision making, leadership, and communication plans.

|
| **Ch. 12** | Software Quality | ISO 9126, process capability models, and quality plans.

|
| **14** | Presentation | Student project presentation.

|

## 5. Course Materials

The primary textbooks recommended for this course are:

- **Primary:** _Software Project Management_ (5th Ed), Bob Hughes and Mike Cotterell, 2008.

- **Secondary:** _Software Project Management in Practice_, Pankaj Jalote, 2002.

- **Secondary:** _Project Management_, Gary R. Heerkens, 2002.

- **Secondary:** _Effective Project Management_, Robert Wysocki & Rudd McGary, 3rd Ed, 2003.

---

## 6. Student Expectations

- **Preparation:** Read materials carefully before attending class and follow case study analyses.

- **Technical:** Students must have a computer to use management software like **Microsoft Project** or **DevOps**.

- **Participation:** Actively participate in group reports and submit assignments on schedule.
